### **Basic input directory**

`sf-tractomics` only needs diffusion MRI images. The input structure should look like this:

<FileTree>
  * input\_directory
    * sub-01
      * sub-01\_dwi.nii.gz
      * sub-01\_dwi.bval
      * sub-01\_dwi.bvec
    * sub-02
      * sub-02\_dwi.nii.gz
      * sub-02\_dwi.bval
      * sub-02\_dwi.bvec
</FileTree>

### **BIDS input directory** (In development)

The [BIDS (Brain Imaging Data Structure)](https://bids-specification.readthedocs.io/en/stable/) input directory is a standardized way to organize and describe neuroimaging and behavioral data. The sf-tractomics pipeline expects the input data to be organized in the BIDS format. Good resources to convert your raw DICOMs files in a BIDS structure include [dcm2bids](https://unfmontreal.github.io/Dcm2Bids/3.2.0/) and/or [heudiconv](https://heudiconv.readthedocs.io/en/latest/). It is recommended that users validate their BIDS layout using the official [bids-validator tool](https://hub.docker.com/r/bids/validator) docker image or the [web-based version](https://bids-standard.github.io/bids-validator/).

`sf-tractomics` will soon be BIDS compatible.

### **If you want to use your own mask**

`sf-tractomics` allows to extract the brain (step 3) with a nnUNet algorithm. If you don't want to use this method, you also can provide a mask, but the input structure should look like this:

<FileTree>
  * input\_directory
    * sub-01
      * sub-01\_dwi.bval
      * sub-01\_dwi.bvec
      * sub-01\_dwi.nii.gz
      * sub-01\_mask.nii.gz
    * sub-02
      * sub-02\_dwi.bval
      * sub-02\_dwi.bvec
      * sub-02\_dwi.nii.gz
      * sub-02\_mask.nii.gz
</FileTree>
